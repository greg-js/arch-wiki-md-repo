### Пользователи и группы

В свежей установке присутствует лишь один аккаунт суперпользователя, более известный как "root". Вход в систему от имени суперпользователя на длительное время, даже на сервер через [SSH](/index.php/Secure_Shell_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Secure Shell (Русский)"), [считается небезопасным](https://apple.stackexchange.com/questions/192365/is-it-ok-to-use-the-root-user-as-a-normal-user/192422#192422). Вместо этого для большинства задач вам необходимо создать и использовать аккаунты непривилегированных пользователей, используя аккаунт суперпользователя только для администрирования системы. Для получения дополнительной информации смотрите раздел [Пользователи и группы#Управление пользователями](/index.php/%D0%9F%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D0%B8_%D0%B8_%D0%B3%D1%80%D1%83%D0%BF%D0%BF%D1%8B#.D0.A3.D0.BF.D1.80.D0.B0.D0.B2.D0.BB.D0.B5.D0.BD.D0.B8.D0.B5_.D0.BF.D0.BE.D0.BB.D1.8C.D0.B7.D0.BE.D0.B2.D0.B0.D1.82.D0.B5.D0.BB.D1.8F.D0.BC.D0.B8 "Пользователи и группы").

Пользователи и группы - это механизм _контроля доступа_. Администраторы могут тонко настраивать владельцев и членство в группах, чтобы предоставлять или запрещать пользователям и службам доступ к системным ресурсам. Для получения дополнительной информации об этом, а также о потенциальных рисках, связанных с безопасностью, читайте статью [Пользователи и группы](/index.php/Users_and_Groups_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Users and Groups (Русский)").

### Эскалация привилегий

Команда [su](/index.php/Su "Su") (substitute user - замена пользователя) позволяет выступить в роли другого пользователя системы (обычно суперпользователя), не завершая текущий сеанс, в то время как команда [sudo](/index.php/Sudo_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Sudo (Русский)") предоставляет временную эскалацию привилегий для конкретной команды.

### Управление службами

В качестве программы [инициализации](/index.php/Init "Init") в Arch Linux используется [systemd](/index.php/Systemd_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Systemd (Русский)"), являющаяся программой управления системой и службами для Linux. Для работы с установленным у вас Arch Linux желательно изучить основы ее использования. Взаимодействие с _systemd_ осуществляется при помощи команды _systemctl_. Для получения дополнительной информации читайте раздел [systemd (Русский)#Основы использования systemctl](/index.php/Systemd_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9)#.D0.9E.D1.81.D0.BD.D0.BE.D0.B2.D1.8B_.D0.B8.D1.81.D0.BF.D0.BE.D0.BB.D1.8C.D0.B7.D0.BE.D0.B2.D0.B0.D0.BD.D0.B8.D1.8F_systemctl "Systemd (Русский)").

### Поддержка системы

Arch - система с плавающими обновлениями (rolling release), и пакеты в ней обновляются довольно часто, так что пользователи должны уделять некоторое время [поддержке системы](/index.php/System_maintenance "System maintenance"). Для получения рекомендаций по тому, как сделать ее более "стойкой", обратитесь к странице [Безопасность](/index.php/Security_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Security (Русский)").

### Система сборки для Arch (Arch Build System)

_Порты_ - это система, изначально использовавшаяся в дистрибутивах BSD и состоящая из скриптов сборки, которые находятся в дереве каталогов локальной системы. Проще говоря, каждый порт содержит скрипт в каталоге с интуитивно понятным названием после устанавливаемого приложения третьей стороны.

[ABS](/index.php/Arch_Build_System_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Arch Build System (Русский)") предлагает тот же функционал, предоставляя скрипты сборки, называемые [PKGBUILD](/index.php/PKGBUILD_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "PKGBUILD (Русский)") и содержащие информацию о месте для программного обеспечения, хэши целостности, URL-адрес проекта, версию, лицензию и инструкции по сборке. Эти файлы PKGBUILD позже анализирует [makepkg](/index.php/Makepkg_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Makepkg (Русский)"), программа, генерирующая пакеты, которыми легко может управлять _pacman_.

Каждый пакет в репозиториях, включая те, что представлены в AUR, подлежит компиляции при помощи _makepkg_.

### Arch User Repository

ABS дает возможность сборки программного обеспечения, доступного в официальных репозиториях. Пользовательский репозиторий Arch — [Arch User Repository (AUR)](/index.php/Arch_User_Repository_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Arch User Repository (Русский)") — их эквивалент для пакетов, предоставляемых пользователями. Это неподдерживаемый репозиторий, содержащий скрипты для сборки и доступный через [веб-интерфейс](https://aur.archlinux.org/) или при помощи [вспомогательных инструментов для AUR](/index.php/AUR_helpers_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "AUR helpers (Русский)").

### Автоопределение оборудования

По умолчанию оборудование определяется автоматически во время загрузки при помощи [udev](/index.php/Udev_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Udev (Русский)"). Потенциально можно уменьшить время загрузки, отключив автозагрузку модулей и указав список необходимых модулей вручную, как описано в статье [Модули ядра](/index.php/%D0%9C%D0%BE%D0%B4%D1%83%D0%BB%D0%B8_%D1%8F%D0%B4%D1%80%D0%B0 "Модули ядра"). [Xorg](/index.php/Xorg_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Xorg (Русский)") также должен уметь определять требуемые драйверы при помощи udev, но у пользователей, опять же, есть возможность настроить X-сервер вручную.

После окончания процесса загрузки экран очищается, и выводится приглашение на ввод имени пользователя, что не дает возможности просмотреть сообщения процесса загрузки. Чтобы изменить такое поведение, [отключите очистку загрузочных сообщений](/index.php/Disable_clearing_of_boot_messages "Disable clearing of boot messages").

Графический драйвер по умолчанию, _vesa_, будет работать с большинством видеокарт, но можно значительно повысить производительность и получить дополнительный функционал, установив соответствующий драйвер для продукции [ATI](/index.php/ATI_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "ATI (Русский)"), [Intel](/index.php/Intel_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Intel (Русский)") или [NVIDIA](/index.php/NVIDIA_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "NVIDIA (Русский)").

### Графический сервер

[Xorg](/index.php/Xorg_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Xorg (Русский)") - это свободная реализация [оконной системы X](https://en.wikipedia.org/wiki/ru:X_Window_System "wikipedia:ru:X Window System") (называемой обычно X11 или X) с открытым исходным кодом. Она необходима для запуска приложений с графическим пользовательским интерфейсом (GUI), и большинство пользователей захочет ее установить.

[Wayland](/index.php/Wayland "Wayland") - новый, альтернативный протокол графического сервера, для которого доступна его эталонная реализация "Weston". На данном раннем этапе разработки он поддерживается приложениями очень слабо.

### Оконные менеджеры

Зрелое окружение рабочего стола предоставляет завершенный и согласованный графический пользовательский интерфейс, но, как правило, потребляет значительное количество системных ресурсов. Пользователи, желающие получить максимальную производительность или упростить окружение, могут выбрать установку [оконного менеджера](/index.php/Window_manager_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Window manager (Русский)") и ручной выбор желаемых дополнений. Альтернативный оконный менеджер можно также использовать с большинством окружений рабочего стола. [Динамические](/index.php/Category:Dynamic_WMs_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Category:Dynamic WMs (Русский)"), [стековые](/index.php/Category:Stacking_WMs_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Category:Stacking WMs (Русский)") и [тайловые](/index.php/Category:Tiling_WMs_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Category:Tiling WMs (Русский)") оконные менеджеры отличаются в том, как они управляют окнами.

### Экранный менеджер

### Окружения рабочего стола

Xorg предоставляет основной фреймворк для построения графического окружения, но есть дополнительные компоненты, которые могут оказаться необходимыми для полноценной работы пользователя. Настройка цветного вывода таких специфичных базовых утилит, как _grep_ и _ls_, описана в статье [Базовые утилиты](/index.php/%D0%91%D0%B0%D0%B7%D0%BE%D0%B2%D1%8B%D0%B5_%D1%83%D1%82%D0%B8%D0%BB%D0%B8%D1%82%D1%8B "Базовые утилиты").

Страницы справочных руководств относятся к числу наиболее полезных ресурсов, доступных пользователям GNU/Linux. Для улучшения читабельности можно настроить цветной вывод текста, как это описано в разделе [Цветные man-страницы](/index.php/Man_page_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9)#.D0.A6.D0.B2.D0.B5.D1.82.D0.BD.D1.8B.D0.B5_man-.D1.81.D1.82.D1.80.D0.B0.D0.BD.D0.B8.D1.86.D1.8B "Man page (Русский)").

### Сжатые файлы

Сжатые файлы или архивы часто используются в системах GNU/Linux. [Tar](/index.php/Tar "Tar") является одним из самых используемых инструментов архивации, так что пользователям следует ознакомиться с его синтаксисом (например, пакеты Arch Linux представляют собой простые архивы, упакованные при помощи _xzip_). Для получения информации о других полезных командах смотрите статью [Bash (Русский)](/index.php/Bash_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Bash (Русский)").

### Приглашение командной строки

Приглашение командной строки (PS1) в значительной степени можно настроить под себя. Если вы используете Bash или Zsh, смотрите, соответственно, статью [Цветное приглашение Bash](/index.php/Color_Bash_Prompt_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Color Bash Prompt (Русский)") или раздел [Цветное приглашение Zsh](/index.php/Zsh_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9)#.D0.9D.D0.B0.D1.81.D1.82.D1.80.D0.BE.D0.B9.D0.BA.D0.B0_.D1.81.D1.82.D1.80.D0.BE.D0.BA.D0.B8_.D0.BF.D1.80.D0.B8.D0.B3.D0.BB.D0.B0.D1.88.D0.B5.D0.BD.D0.B8.D1.8F_.28PROMPT.29 "Zsh (Русский)").

### Оболочка Emacs

Emacs хорошо известен своим широким функционалом, распространяющимся далеко за пределы обычного редактирования текста, благодаря чему может полностью заменить оболочку. Для исправления проблем с искажением символов при цветном выводе обратитесь к разделу [Проблемы с цветным выводом](/index.php/Emacs#Colored_output_issues "Emacs").

### Поддержка мыши

Использование мыши для копирования и вставки текста в консоли может быть более предпочтительным, чем использование традиционных методов [GNU screen](/index.php/GNU_Screen_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "GNU Screen (Русский)"). Для получения точных инструкций обратитесь к статье [Поддержка мыши в консоли](/index.php/%D0%9F%D0%BE%D0%B4%D0%B4%D0%B5%D1%80%D0%B6%D0%BA%D0%B0_%D0%BC%D1%8B%D1%88%D0%B8_%D0%B2_%D0%BA%D0%BE%D0%BD%D1%81%D0%BE%D0%BB%D0%B8 "Поддержка мыши в консоли").

### Буфер прокрутки

Для получения возможности сохранять и просматривать текст, который при прокрутке ушел за пределы экрана, обратитесь к статье [Буфер прокрутки](/index.php/Scrollback_buffer "Scrollback buffer").

### Управление сессиями

При использовании терминальных мультиплексоров, таких как [tmux](/index.php/Tmux_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Tmux (Русский)") или [GNU Screen](/index.php/GNU_Screen_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "GNU Screen (Русский)"), программы могут запускаться в сессиях, состоящих из вкладок и панелей, которые по желанию можно отделять с тем, чтобы, когда пользователь закрывает эмулятор терминала, выходит из [X](/index.php/Xorg_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Xorg (Русский)") или сеанса, программы, связанные с сессией, продолжали работать в фоновом режиме, пока активен сервер терминального мультиплексора. При этом для взаимодействия с программами потребуется повторный вход в сессию.
