Официальный сайт [I2P](https://geti2p.net/ru/)

	I2P (аббревиатура от англ. invisible internet project, IIP, I2P — проект «Невидимый интернет») — проект, начатый с целью создания анонимной компьютерной сети, работающей поверх сети интернет. Она предназначена для защиты передачи данных от внешнего наблюдения и надзора, например, от слежки провайдера.

	Внутри сети I2P можно разместить любой сервис (или службу) (форум, блог, файлообменник, электронную почту, систему для мгновенного обмена сообщениями (чат), систему для совместного использования файлов, VoIP и т. д.) с сохранением анонимности сервера. В сети I2P работают http-серверы; адреса сайтов находятся в псевдо-доменном пространстве «.i2p». Поверх сети I2P можно строить одноранговые сети (P2P), например, BitTorrent, eDonkey, Kad, Gnutella и т. д.

	Сеть I2P никоим образом не имеет отношения к проекту Tor - это две совершенно разные технологии. Tor это скорее цепочка прокси серверов, тогда как I2P это полноценная сеть. Сеть I2P это практически тот же самый интернет, к которому все привыкли. Там тоже есть сайты, торренты, почта, и прочее. Только всё это анонимно и зашифровано от посторонних, а также не поддаётся контролю третьими сторонами. Т.е. там нет цензуры, нет каких-либо ограничений на передаваемую информацию. И нет возможности отследить – ЧТО именно вы просматриваете в этом, новом интернете, какие сайты посещаете, что качаете, какой ваш круг интересов, знакомств и т.д.

	Так же важно помнить что в I2P сети нет привычных DNS серверов и сеть абсолютно не зависит от обычных DNS и каких либо других серверов, что приводит к невозможности уничтожения, блокирования и фильтрации сети - сеть I2P будет существовать и функционировать пока на планете останутся хотя бы два компьютера в сети.

## Установка

I2P доступен с пакетом [i2p](https://aur.archlinux.org/packages/i2p/), предоставляющим компиляцию из исходного кода, и пакет [i2p-bin](https://aur.archlinux.org/packages/i2p-bin/) предоставляет предварительно скомпилированный двоичный файл.

Ставим i2p командой

```
yaourt -S i2p 

```

Добавляем gpg ключ, просто введя в терминал строку ниже

```
gpg --keyserver pgp.mit.edu --recv-keys 85F345DD59683006

```

## Запуск и настройка

Сначала запустите `i2prouter.service`. Это запустит [демон](/index.php/Daemons_(%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9) "Daemons (Русский)") i2p. Затем откройте ваш браузер и перейдите на страницу приветствия I2P по адресу:

```
127.0.0.1:7657/home

```

После этого вы можете перейти к страницам конфигурации и статистики I2Ps и к каталогу ссылок eepsite (eepsite - это сайт, доступный только через сеть I2P, подобно тому, как сайты .onion доступны только через сеть Tor). Кроме того, имейте в виду, что eepsites недоступны пока демон не загрузится в сеть, что может занять несколько минут.

Переходим на страницу:

```
[http://127.0.0.1:7657/confignet](http://127.0.0.1:7657/confignet)

```

Находим UPnP Configuration, ставим напротив галочку и сохраняем. UPnP (Universal Plug and Play) позволяет автоматически открывать порт при запросе программы. В результате, мы должны получить статус сети "Ок" Если статус другой, то смотрите раздел помощь.

```
[http://127.0.0.1:7657/confignet#help](http://127.0.0.1:7657/confignet#help)

```

Переходим в закладку "Ограничение скорости" или "Bandwidth limiter" и ставим настройки входящего и исходящего трафика. (Например, входящий трафик 512 и исходящий 128)

Чтобы включить использование прокси, используйте следующие настройки браузера:

```
HTTP  127.0.0.1 4444
HTTPS 127.0.0.1 4445

```

Для настройки прокси в браузере Firefox переходим в Настройки -> Дополнительные -> Ручная настройка сервиса прокси.

При необходимости отключите firewall (или настройте, чтобы он не блокировал сеть)

```
sudo ufw disable

```

## Команды для управления i2p роутером

```
systemctl start i2prouter.service - запуск
systemctl stop i2prouter.service - остановка
systemctl restart i2prouter.service - перезапуск
systemctl status i2prouter.service - показывает статус вкл./выкл

```

## Создание сайта в анонимной сети

Если вы создаете eepsite, следуйте инструкциям I2P, но имейте в виду, что домашний каталог будет применяться к пользователю i2p, домашняя директория будет `/opt/i2p` Как показано в AUR [i2p.install](https://aur.archlinux.org/cgit/aur.git/tree/i2p.install?h=i2p)
