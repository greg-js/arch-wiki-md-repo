## Введение

В этом документе описывается установка и настройка ArchLinux на ноутбуке RoverBook Voyager H591

## Описание компьютера

Ноутбук с процессором Intel Celeron M, 1.5 GHz, 512 RAM, Video Via Unichrome VN800 64 Mb (использует основную память) 1024х768, HDD 40Gb, DVD-RW (хотя на самом приводе написано что это комбинированный девайс CD-RW/DVD-ROM, видимо у изготовителей не нашлось нужной крышки для лотка :). Приобретён в 2006 году за 800$. Хорошая тачка, надёжная и быстрая. Примечательно полное отсутствие нестандартного оборудования (WiFi, IR, bluetooth, камера). Если бы сегодня продавали что-то подобное, только помощнее и с предустановленным Arch...(это пожелание компании Rover Computers).

## Инсталляция

Без проблем, компьютер входит в домашнюю сеть через маршрутизатор на Windows XP (а почему не наоборот!), указываются фиксированные ip адреса. Разбиение диска произведено заранее: 500 sda1 (swap), 9500 sda2 (root), 20000 sda3 (home), 10000 sda4 (fat), загрузчик - lilo (требуется некоторая настройка файла lilo.conf). Grub я не устанавливаю по одной простой причине - до него не дошли руки, а ставить загрузчик, о котором ты имеешь лишь смутное представление не самый лучший вариант.

## Конфигурирование

Всё оборудование работоспособно (модули ядра загружаются автоматически). Драйверы для видеокарты выбираются из 3-х вариантов: xf86-video-via, xf86-video-unichrome, openchome:

Первый вариант: проблемы с производительностью видео в некоторых приложениях, долгий запуск xorg.

Второй вариант: не поддерживается чипсет.

Третий вариант: есть некоторые глюки, но это устранимо.

Установка третьего драйвера из репозитория:

```
pacman -S openchrome

```

Далее - настройка файла xorg.conf проживающего по адресу:

```
vim /etc/X11/xorg.conf

```

Сначала в секции Device выбираем наш драйвер строкой:

```
Driver     "openchrome"

```

Как я уже говорил с ним могут быть некторые проблемы. В частности при настройках по умолчанию у меня происходили необъяснимые затемнения экрана (подобно тому как это было в Windows 95 после т.н синей смерти) От этого можно избавится, вставив следующую строку в секцию Device:

```
Option     "NoAGPFor2D" 

```

Насколько это влияет на поведение драйвера я не знаю, но вроде работает. Кроме того неплохо добавить ускорение EXA в ту же секцию строчкой:

```
Option     "AccelMethod"   "EXA"

```

И некоторые необязательные опции:

```
Option     "PanelSize"     "1024x768"
Option     "ActiveDevice"  "LCD"

```

Далее, чтобы настроить нужное разрешение, я отредактировал секцию Screen подобным образом:

```
Section "Screen"
       Identifier "Screen0"
       Device     "Card0"
       Monitor    "Monitor0"
       DefaultDepth    32
       SubSection "Display"
               Viewport   0 0
               Depth   32
               Modes   "1024x768"
       EndSubSection
EndSection

```

То есть фиксируем одно разрешение 1024x768 при 32 битном цвете. Кому нужно больше - карты в руки. Вот вроде и всё с настройкой специфичных вещей.

## Послесловие

Arch Linux отлично подходит для установки на ноутбук, заменяет windows на 99%, а для всего остального существуют виртуальные машины.
